## W1-① 구독 상품 구조 확정

### 플랜 설계 (v1은 2단 + 무료)

**Free**
- 해몽 2회/일
- 기본 꿈 기록
- 커뮤니티 열람
- 달이 기본 대화 (제한)

**Plus**
- 해몽 사실상 무제한
- 달이 대화 확장
- 주간 리포트
- 광고/잠금 제거

**Premium**
- 반복꿈 패턴 분석
- 감정 변화 추적 리포트
- 장기 아카이브 인사이트
- 고급 운세/꿈 사전/심볼 리포트

### 왜 2단인가
- 이 앱은 "기능 도구"가 아니라 "정서적 루틴 앱"
- 3~4단으로 쪼개면 결제 피로만 증가
- 초기 목적: 전환율 측정, 해지 방지, 심사 단순화

### 스토어별 SKU 매핑

```
내부 상품 키           │ Web (Stripe)          │ iOS (App Store)                  │ Android (Play)
───────────────────────┼───────────────────────┼──────────────────────────────────┼─────────────────────────
monggeul_plus_monthly  │ price_xxxxx           │ com.monggeul.plus.monthly        │ monggeul_plus / monthly
monggeul_premium_monthly│ price_yyyyy          │ com.monggeul.premium.monthly     │ monggeul_premium / monthly
```

### v1에서 하지 말 것
- 연간 요금제, 가족 플랜, 쿠폰/프로모션
- 지역별 가격 실험, 웹↔앱 업셀 꼼수
- 복잡한 애드온

### v1에서 반드시 할 것
- 월 구독 2개 (Plus / Premium)
- 복원 버튼 (Apple 필수, 없으면 리젝)
- 서버 검증
- 실시간 상태 반영
- 페이월 문구 2종 (A/B)
- 전환 로그 수집

## W1-② Supabase entitlement 스키마

### Claude Code 명령어

```bash
# Supabase SQL Editor 또는 마이그레이션 파일로 실행

cat > supabase/migrations/20260321_billing_schema.sql << 'EOSQL'

-- ============================================
-- 1) 사용자 권한 상태 (UI가 바라보는 유일한 테이블)
-- ============================================
create table public.user_entitlements (
  user_id              uuid primary key references auth.users(id) on delete cascade,
  entitlement_key      text not null default 'free',
    -- free / plus_active / premium_active / grace_or_hold
  source_platform      text,
    -- web / ios / android
  product_key          text,
    -- monggeul_plus_monthly / monggeul_premium_monthly
  status               text not null default 'inactive',
    -- active / grace / hold / expired / canceled / refunded
  current_period_start timestamptz,
  current_period_end   timestamptz,
  will_renew           boolean,
  auto_renew           boolean,
  last_verified_at     timestamptz,
  updated_at           timestamptz not null default now()
);

-- RLS: 본인만 읽기
alter table public.user_entitlements enable row level security;
create policy "Users read own entitlement"
  on public.user_entitlements for select
  using (auth.uid() = user_id);

-- ============================================
-- 2) 원본 거래 로그 (감사/디버깅/분쟁 대응)
-- ============================================
create table public.billing_transactions (
  id                   bigint generated always as identity primary key,
  user_id              uuid references auth.users(id) on delete cascade,
  platform             text not null,
    -- stripe / apple / google
  platform_account_ref text,
    -- Stripe customer_id / Apple originalTransactionId / Google purchaseToken
  product_key          text not null,
  transaction_ref      text not null,
    -- Stripe invoice_id / Apple transaction_id / Google order_id
  event_type           text not null,
    -- purchased / renewed / failed / expired / refunded / revoked
  amount               numeric(12,2),
  currency             text,
  raw_payload          jsonb not null,
  occurred_at          timestamptz not null,
  created_at           timestamptz not null default now()
);

-- ============================================
-- 3) 웹훅/알림 멱등성 (중복 수신 방지)
-- ============================================
create table public.billing_events (
  event_id    text primary key,
  platform    text not null,
  event_type  text not null,
  payload     jsonb not null,
  processed   boolean not null default false,
  processed_at timestamptz,
  created_at  timestamptz not null default now()
);

EOSQL

echo "✅ 빌링 스키마 SQL 생성 완료"
echo "→ Supabase Dashboard > SQL Editor에 붙여넣거나"
echo "→ supabase db push로 마이그레이션 실행"
```

### 설계 원칙
- **user_entitlements** = 현재 상태 (앱/API가 이것만 봄)
- **billing_transactions** = 감사 로그 (회계·분쟁 대응)
- **billing_events** = 멱등성 보장 (웹훅 중복 처리 방지)

### 권한 상태 머신

```
               ┌─ 최초 구매 / 갱신 성공 / 업그레이드 ─┐
               ▼                                       │
  [free] ──→ [plus_active / premium_active] ──→ [grace_or_hold]
               │                                       │
               │   ← 유예 복구 / 결제 재시도 성공 ←──┘
               │
               ├─ 유예 종료 / 자동갱신 off + 기간만료 ──→ [expired] → [free]
               └─ 환불 / revoke ──→ 즉시 [free] + abuse 플래그
```

```bash

cat > supabase/functions/billing-apple-verify/index.ts << 'EOF'
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

  //    GET https://api.storekit.itunes.apple.com/inApps/v1/transactions/{transactionId}

  // 3. user_entitlements 갱신:

  //    status = 'active'
  //    source_platform = 'ios'
  //    current_period_start / end 설정

  // 4. 응답
  return new Response(JSON.stringify({ entitlement: "plus_active" }), {

  });

EOF
```

## ⚠️ 심사 리젝 방지 체크리스트

1. **Apple IAP 필수** — 디지털 콘텐츠(해몽)이므로 IAP 외 결제수단 사용 시 리젝
2. **구매 복원 버튼** — 없으면 리젝 (Apple 가이드라인 3.1.1)
3. **개인정보 처리방침 URL** — 두 스토어 모두 필수
4. **면책 문구** — "오락/자기성찰 목적이며 의학적 진단이 아님" (운세/건강운 관련)
5. **AI 콘텐츠 고지** — App Store가 AI 생성 콘텐츠 고지를 요구할 수 있음
6. **커뮤니티 봇** — 자동 생성 게시물이 "실제 사용자"로 오인되지 않도록 주의
7. **앱명 30자 제한** — Apple 메타데이터 규칙
8. **스크린샷에 가격 표시 금지** — Apple 가이드라인
9. **Google purchaseToken acknowledgment** — 안 하면 3일 후 자동 환불
10. **한국 Google Play 대체결제** — v1은 Play Billing 단일이 안전 (속도 우선)
